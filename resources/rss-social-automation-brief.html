Safe scope

Separate draft generation from live posting permissions.

01

Start with drafts.

AI summaries and draft posts are usually safer to test before live auto-posting is enabled.

02

Confirm feed rights.

The buyer must own or have permission to summarize and republish the feed content.

03

Name approval rules.

List who reviews drafts, what can be posted, and when a draft must be held for manual review.

04

Avoid platform bypasses.

Any live posting path must use approved tools, connected accounts, or platform-compliant APIs.

Workflow map

A clear first build has five controlled steps.

Feed

Read new items.

Check the feed on a schedule and skip duplicate URLs or unchanged entries.

AI

Summarize safely.

Generate concise summaries using the buyer's tone, platform, avoid-list, and link rules.

Draft

Create post options.

Produce social drafts with hooks, short captions, hashtags if allowed, and source links if needed.

Review

Hold for approval.

Route drafts to email, Slack, Notion, Airtable, Sheets, or another agreed review location.

Post

Post only if scoped.

Live posting is separate and should only be enabled after a small test and platform-safe setup.

Log

Track outputs.

Store generated drafts, approvals, skipped items, errors, and next run time.
